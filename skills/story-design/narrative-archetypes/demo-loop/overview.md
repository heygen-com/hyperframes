# Demo Loop — Question → Instant Answer

## Core Logic

Minimal narrative setup → rapid product reveal → repeated demonstration cycles that escalate in scope. The product's UI and output quality _are_ the persuasion. Narrative takes a backseat to "seeing is believing."

## Emotional Arc Pattern

```
Urgency ──▶ Relief ──▶ Rhythm (demo cycles) ──▶ Confidence ──▶ Action
  (hook)   (reveal)    (proof 1→2→3)          (trust)        (CTA)
```

## Typical Section Sequence

| Order | Type              | Role                                      | Approx % |
| ----- | ----------------- | ----------------------------------------- | -------- |
| 1     | hook              | Quick pain/urgency question               | 5-8%     |
| 2     | product_intro     | Name the solution immediately             | 5-8%     |
| 3     | feature_showcase  | Demo cycle 1: specific niche query        | 15-20%   |
| 4     | feature_showcase  | Demo cycle 2: different domain (breadth)  | 15-20%   |
| 5     | benefit_highlight | Demo cycle 3: strategic use + trust claim | 25-30%   |
| 6     | cta               | Tagline + next step                       | 10-15%   |

## When to Use

- The product's UI is visually compelling and self-explanatory
- "Show, don't tell" is more convincing than emotional storytelling
- Data tools, search tools, dashboards — where the output speaks for itself
- The value proposition is simple: ask → get answer

## When NOT to Use

- The product solves a complex emotional pain that needs setup (→ use PAS)
- The product is creating a new category that needs vision-selling (→ use Future Pacing)
- The UI is not yet polished or the output is hard to show visually
